Tutorial (tutorial.typ)

Standard library

  • prelude.typ: simple, common types and function (booleans, option, ...)
  • nat.typ: unary natural numbers
  • list.typ: usual list type
  • applist.typ: supertype of lists with constant time concatenation
  • set.typ: set with unbalanced binary search tree
  • natbin.typ: binary representation of natural numbers
  • stream.typ: streams (or infinite lists)
  • state_stream.typ: alternative representation of streams, using an internal state
  • state_array.typ: state monad with an association list as state
  • tree23.typ: maps implemented using 2-3 trees
  • int.typ: unary representation of integers (supertype of nat, no trailing zero)
  • real.typ: signed digit representation of real numbers (following Alex Simpson)

Sorting functions on lists

  • insert_sort.typ: size-preserving insertion sort
  • quick_sort.typ: quicksort function
  • heap_sort.typ: heapsort function

Church encoding

  • church/bool.typ: Church-encoded booleans
  • church/nat.typ: Church naturals
  • church/data.typ: Church-encoded sums and products
  • church/list.typ: Church-encoded lists
  • church/error.typ: Church-encoded error monad (option type)
  • church/state.typ: Church-encoded state monad
  • church/stream.typ: Church-encoded streams (infinite lists)
  • church/david.typ: infimum on Church numeral by René David

Scott encoding

  • scott/nat.typ: Scott-encoded natural numbers
  • scott/list.typ: Scott-encoded lists
  • scott/stream.typ: Scott-encoded streams (inifinte lists)
  • scott/natbin.typ: binary representation of natural numbers
  • scott/tree.typ: Scott-like encoded for binary trees
  • scott/nat_as_prod.typ: other encoding Scott naturals using records

Examples

  • hard.typ: terminating function not satisfying the usual semi-continuity condition
  • lazy_nat.typ: lazy variant of unary natural numbers (interesting for termination)
  • stream_filter.typ: filters on streams (using mixed inductive and coinductive types)
  • ordinal.typ: type of ordinals with specific addition functions
  • lambda.typ: λ-calculus using parapetric higher-order abstract syntax
  • simply.typ: simply-typed λ-calculus
  • tree.typ: red-black trees as a subtype of binary trees
  • size.typ: various tests with sized type and termination
  • dotproj.typ
  • dotprojeps.typ
  • polyrec.typ
  • permutte.typ
  • failterm.typ
